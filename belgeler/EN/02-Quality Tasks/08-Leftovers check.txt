Although leftovers checks seem like something done at the end of the day, they should actually be started at the beginning of the day as well, because even if everything is done correctly in our shift, things missed by the previous shift can cause us to encounter leftovers.
When I start my day, I do a PNOV check just like the night shift does, but only for oversized packages, because they will remain on the OV racks as leftovers. SCC -> Sort -> Stow -> Stow Breakdown -> Click on the numbers for Inducted (EingefÃ¼hrt) packages -> Check the Size -> Only search for Large or XXL
In the Problem Solve-Task section of TC, you'll start seeing the "Aisles sweeping" notification when all the items in those aisles have been taken. Once you see this notification, you should go to the aisle and check if any items are left.
When we find leftovers we should scan them with our TC (Problem Solve-Overview).
-If the status STOWED, STOWED(without Route code), PICK UP FAILED; Driver Problem -> Pick Left Behind -> Continue Solving -> Cluster Transfer -> Solve more problems
-If the status IN TRANSIT, MARKED AS MISSING; Dolphin Device->Receive->Receive packages
-If status STOW BUFFERED, INDUCTED; scan with Problem Solve-Task, it will ask you to stow it. After you stow it will be done. You can also give these parcels to DAs for manually pick up.
* At the start of my shift, I quickly visually scan each aisle to identify any problems before they arise. We also need to check the bags on the shelves that aren't being used for stow by NS, because sometimes they contain packages belonging to other bags.
** When we find a leftover bag, we need to check its history using SCC -> Package Summary -> Search to determine why it's leftover. We should also check the bag's route code to see if it's included in the picklist. If it is, the problem is due to a picker error; otherwise, it's because someone on the night shift scrubbed the bag, causing the system to remove it from the picklist.
